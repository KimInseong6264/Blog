# Ⅱ. object & 박싱,언박싱

### 1. 모든 자료형의 근간 object
---
**※object:** 변수 지정을 할 경우 어떠한 자료형도 넣을 수 있는 만능 자료형
- 기본 자료형 , 구조체등의 자료형을 넣을 수 있다.
- 입력은 마음대로 넣을 수 있지만 출력할 시에는 들어있는 값에 따라 형변환을 해주어야 한다.
  (오류가 많은 원인)
- **문제점**
  ① 변수를 꺼내 쓸 때, 형변환이 제대로 이루어지지 않는 경우에 터져버린다.
  ② 어떤 자료형인지 시각적으로 구분이 어려워 가독성을 떨어뜨린다.
  ③ 값, 참조 타입을 넘나들면서 메로리 할당, GC등의 이유로 성능 하락
##### 위의 문제점들로 인해 실무에서 사용을 하지 않는다.


박싱(Boxing): stack메모리의 값을 복사하여 Heap 공간에 재할당 하는 것
- 암시적으로(추가 코드 작성x) 사용가능(object타입과 기본 자료형들이 상속관계로 [업캐스팅] 되었기 때문)

언박싱(Unboxing): Heap공간의 값을 복사하여 stack메모리에 재할당 하는 것
- 명시적으로(형변환 필요) 사용가능(object타입과 기본 자료형들이 상속관계로 [다운캐스팅] 되었기 때문)

> **var과의 차이점**
>	object : 자료형 자체를 타입 그대로 담는 자료형
>	var : 필요한 자료형으로 둔갑하는 자료형
