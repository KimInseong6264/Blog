### 1. Git
---
**※Git:** 기존 작성 내용에서 변동사항을 추적, 작업 중인 변경사항을 취소 및 복구하기 위한 세이브 파일 관리툴
- 코드를 작성하며 그 때 그 때 세이브 포인트(파일 전체X)만을 저장함
- 세이브 포인트는 변동사항만 부분적으로 저장하기 때문에 *용량을 효율적으로* 활용가능


### 2. Git의 기능
---
#### 1. 파일의 추적 및 관리
 -지금 당장, 이전 세이브본과 어떤 차이가 있는지 바로 확인 가능

#### 2. 기록 관리
 -세이브 파일간 기록 비교(상세한 내용 기입 가능)

#### 3. 브랜치 관리
 -여러 작업을 분리하고 작업을 병합
 -분기점을 두고 작업 가능
 -세이브 파일을 남겨두고, 루트를 나누어서 진행 가능

#### 4. 협업 및 원격 저장소 기능
 -원격 저장소에 내용을 보내고 받을 수 있음
 -다른 사람도 같은 저장소에 보내고 받을 수 있음

#### 5. 변경사항을 되돌릴 수 있음


### 3. Git의 용어
---
- Repository: 저장소
- local : User의 컴퓨터
- Remote: 원격(GitHub의 클라우드)


![](https://velog.velcdn.com/images/iissk/post/06321258-01fe-46ec-b11b-3c53b1d41720/image.png)

- Working directory : Git의 파일 폴더
	- Git add : 모든 파일의 변동사항을 추적하는 것
- Staging area : 변동사항을 임시로 올려놓는 곳(Git의 주 화면)
	- Git commit : Staging area의 내용을 Localrepo(내 컴퓨터)에 세이브하는 것
- Localrepo : Git commit이 저장되는 내 컴퓨터의 폴더
	-  Git push : 원격 클라우드에 세이브하는 것
- Remote repo : 원격 클라우드 공간 = GitHub 웹클라우드
