# Ⅰ. 객체지향


### 1. 객체지향이란?
---
**※객체지향:** 각 변수, 구조체와 그 기능들을 함수로 따로 구성을 했지만, 데이터(변수,구조체)와 기능을 합쳐서 객체라고 한다.

ex) 게임 자동차 넣을 예정
- 요소: 엔진, 바퀴, 핸들, 브레이크 등등
- 기능: 전진, 후진, 워셔액 분사, 라디오 등등
**=>** 현실의 모든 것들을 담을 필요x, 추상화 진행

**※추상화:** 복잡한 현실의 세부 지향 생략, 문제 해결에 필요한 요소와 기능만 담는 것

**※클래스:** 프로그램 필요한 요소와 기능을 묶어서 설계도화 한 것

**※객체:** 클래스를 기반으로 만들어진 실제 데이터의 동작 결합체( ex. int a; / float b 등의 자료형 부여 포함)
- **필드(매개변수):** 클래스 내에서 정의된 변수(private 기본)
- **메서드:** 특정 객체가 가지는 기능, 성질등
- **인스턴스:** 클래스로 만들었고, new할당이 되어있는 것( Random rd = new Random(); )


### 2. 값 타입과 참조 타입
---
**※struct와 class의 차이**
**구조체:** 크기가 작고 간단한 객체 정의용(16Byte 이하, 변수 4개정도) / **값타입**
	- 데이터 뭉치의 역할이 강하다.
	- 값타입이기에 스택에 잡히고, 빠른 연산 목적

**클래스:** 다소 복잡한 객체를 표현할 때 사용 / **참조타입**
	- 기능과 속성을 묶은 것
	- 행동에 포커스가 맞춰져 있다.
